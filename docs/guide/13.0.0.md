# v13.0.0 澄明

v13.0.0 是 CopyTranslator 的 Tauri 2 迁移优化版本。它保留复制即翻译的核心工作流，并新增或完善 AI 翻译、OCR、透明窗口、快捷键配置、跨平台构建和标准配置目录。

如遇问题，请到 [GitHub Issues](https://github.com/wang-borong/CopyTranslator/issues) 反馈。反馈前请确认正在使用最新的 [GitHub Release](https://github.com/wang-borong/CopyTranslator/releases)。

## 快速上手

1. 启动 CopyTranslator。
2. 确认主界面已启用“监听剪贴板”。
3. 在 PDF、浏览器、编辑器或其他软件中选中要翻译的文本。
4. 执行系统复制操作。
5. CopyTranslator 会自动清理文本、识别语言并显示译文。

如果一段文字被分在多个页面或多个区域中，可以使用“增量复制”。右键布局按钮可标记下一次复制为增量复制，也可以在快捷键设置中绑定“模拟增量复制”。

## 界面与主题

设置入口通常位于主界面左侧。v13 优化了设置页宽度、主界面译文区、小窗口提示和语言显示。

常用设置：

- 设置 -> 外观：切换浅色、深色或跟随系统主题。
- 设置 -> 外观：调整字体、内容内边距、行高和窗口透明度。
- 设置 -> 通用：设置软件语言。
- 设置 -> 快捷键：配置全局快捷键和窗口内快捷键。

透明窗口需要系统窗口合成器支持。Linux 下如果只看到半透明背景但无法透出底层窗口，请先确认当前桌面环境或窗口管理器已启用合成器。

## 翻译引擎

主界面的引擎菜单用于选择当前翻译器。v13 保留常见在线翻译引擎，并补充 AI 翻译和多源对比相关配置。

推荐流程：

1. 打开设置 -> 翻译器管理。
2. 启用需要使用的引擎。
3. 如引擎需要密钥，在对应配置项中填写 API Key、Secret 或 App ID。
4. 将常用引擎加入默认分组、多源对比分组或缓存分组。
5. 回到主界面选择翻译引擎。

如果某个引擎不支持当前语言，CopyTranslator 会尝试使用备用引擎。备用引擎可在设置中调整。

## AI 翻译

AI 翻译基于 OpenAI 兼容的 Chat Completion API。OpenAI、DeepSeek、Moonshot、智谱 AI、阿里云百炼、Ollama、NVIDIA NIM、OpenRouter 以及自定义端点都可以通过同一套配置接入。

添加 AI 供应商：

1. 打开设置 -> 自定义翻译器。
2. 点击“添加 AI 供应商”。
3. 选择供应商模板，或选择“自定义”。
4. 填写供应商名称、API Base URL 和 API Key。
5. 点击“刷新模型”获取模型列表。
6. 勾选要启用的模型。
7. 保存配置。
8. 回到主界面，在翻译引擎菜单中选择对应模型。

高级配置：

- 提示词预设：选择忠实、自然、技术、学术、日常或自定义风格。
- 翻译角色：定义模型扮演的翻译角色，例如技术文档译者、论文润色译者。
- System Prompt：完全自定义系统提示词。
- 温度：数值越低越稳定，数值越高越自由。
- 最大 Token：限制单次输出长度。
- 保留格式：尽量保留段落、列表、代码块和换行结构。

建议从低温度和“忠实”预设开始调试，确认结果稳定后再增加风格化提示词。

## OCR

v13 集成百度 OCR REST API。OCR 用于识别剪贴板中的图片文字，然后交给当前翻译引擎翻译。

使用步骤：

1. 打开设置 -> OCR。
2. 启用 OCR。
3. 在 `baidu-ocr` 配置中填写 `app_id`、`api_key` 和 `secret_key`。
4. 使用系统截图工具截图，并将图片复制到剪贴板。
5. CopyTranslator 检测到图片后会识别文字并翻译。

OCR 会把图片内容发送到所配置的 OCR 服务。处理合同、论文原稿、公司文档或其他敏感内容前，请先确认有权上传。

## 快捷键

v13 可以在设置 -> 快捷键中配置快捷键。

默认全局快捷键：

| 动作 | 快捷键 |
| --- | --- |
| 专注模式 | `Shift+F1` |
| 对照模式 | `Shift+F2` |
| 模拟复制 | `Super+Backquote` |
| 模拟增量复制 | `Super+Shift+Backquote` |

默认窗口内快捷键：

| 动作 | 快捷键 |
| --- | --- |
| 复制译文 | `CmdOrCtrl+S` |
| 复制原文 | `CmdOrCtrl+D` |
| 隐藏窗口 | `Escape` |

如果保存后快捷键不生效，通常是系统或其他应用占用了同一组合键。换一个组合键并保存即可。

## 配置目录

CopyTranslator v13 使用系统标准配置目录。

| 系统 | 配置目录 |
| --- | --- |
| Linux | `~/.config/copytranslator` |
| macOS | `~/Library/Application Support/copytranslator` |
| Windows | `%APPDATA%\copytranslator` |

主配置文件为 `copytranslator.json`。首次启动时，程序会尝试从旧路径 `~/copytranslator/copytranslator.json` 迁移。

API Key 会保存在本地配置文件中，请将配置目录视为敏感数据。

## 常见问题

### 主窗口透明但内容不可见

先切换到不透明或较高透明度，确认主题为浅色或深色之一。如果 Linux 桌面环境未启用合成器，透明效果可能无法正确显示底层窗口。

### 翻译引擎没有出现在主界面

进入设置 -> 翻译器管理，确认该引擎已经启用，并且必要的 API 配置已经保存。AI 翻译还需要至少启用一个模型。

### AI 翻译请求失败

检查 API Base URL、API Key、模型权限、账户余额和代理设置。自定义供应商必须兼容 OpenAI Chat Completion API。

### OCR 没有结果

确认 OCR 已启用，百度 OCR 的 `app_id`、`api_key`、`secret_key` 已保存，并且复制到剪贴板的是图片而不是图片文件路径。

### 全局快捷键无法注册

更换快捷键组合。部分系统快捷键、输入法快捷键或窗口管理器快捷键不能被应用覆盖。
