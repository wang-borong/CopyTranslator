# v13 澄明

欢迎在 [GitHub Issues](https://github.com/wang-borong/CopyTranslator/issues) 反馈问题。

## v13.0.4 更新检查与发布包命名

v13.0.4 是 v13 的维护版本，重点增强更新检查可靠性，并统一 Arch Linux 发布包命名。

### 更新检查

- 优先读取 GitHub Releases API 获取最新版本。
- 如果 GitHub API 不可用或被限流，会降级读取 Release 最新版本重定向地址。
- 启动时自动检查更新失败不再弹出错误提示，避免网络波动影响正常使用。

### 发布包

- Arch Linux 发布包命名调整为 `CopyTranslator_13.0.4_x86_64.tar.zst` 风格，与其它 Linux 发布包保持一致。
- 安装包内部仍保留 pacman 可识别的 `.PKGINFO`、desktop 文件、hicolor 图标和 license 文件。

## v13.0.2 置顶、剪贴板与翻译器可用性

v13.0.2 是 v13 的维护版本，主要提升 Tauri 迁移后桌面端窗口行为和剪贴板翻译的可靠性。

### 窗口与剪贴板

- 增强 Linux/KDE/Wayland 下的总是置顶处理，支持 Tauri、GTK、KWin fallback，并在可用时优先使用 XWayland 后端。
- 剪贴板监听改为事件加轮询双通道，降低复制内容偶发漏监听的概率。
- 文本剪贴板会触发翻译；图片剪贴板在 OCR 可用时走 OCR 识别并翻译，OCR 不可用时过滤。
- 清理非文本剪贴板对上一次文本缓存的影响，避免后续复制相同文本时被误判为重复。

### 翻译器

- 修复百度、彩云、腾讯云、Stepfun 等需要密钥的引擎未配置时仍被默认启用的问题。
- 默认主引擎和后备引擎改为更适合作为开箱默认值的 Google 配置。
- Stepfun 支持配置 API Base、API Key 和模型，便于使用官方 OpenAI 兼容接口。
- 腾讯交互翻译增加响应结构兼容处理。

## v13.0.1 快捷键与 Arch Linux 发布包

v13.0.1 是 v13 的维护版本，主要修复输入区快捷键冲突，并为 Arch Linux 用户补充 pacman 可识别的安装包。

### 快捷键修复

- 修复原文输入框中 `Ctrl+Z` / `Cmd+Z` 无法撤销的问题。
- 输入框、textarea、contenteditable 内的撤销、重做、剪切、复制、粘贴、全选优先使用原生编辑行为。
- 保留应用快捷键行为，输入区聚焦时仍可使用复制译文、复制原文、隐藏窗口等应用快捷键。
- 补齐窗口角色类快捷键处理，包括退出、最小化、重载、全屏和缩放。

### 发布包

- Release CI 新增 Arch Linux 安装包。
- 安装包包含 `.PKGINFO`、`/usr/bin/copytranslator`、desktop 文件、hicolor 图标和 license 文件。
- Arch Linux 用户可使用 `sudo pacman -U CopyTranslator_<version>_x86_64.tar.zst` 安装。

## v13.0.0 Tauri 2 迁移优化版

v13.0.0 是迁移优化后的第一个大版本。该版本的目标是彻底摆脱 Electron 运行时，使用 Tauri 2 重建桌面端能力，同时把近期围绕界面、AI 翻译、OCR、快捷键、透明窗口和跨平台发布的修复整理成稳定版本。

### 桌面端迁移

- 完成 Tauri 2 桌面端迁移，移除旧 Electron 运行时入口和相关依赖。
- 使用 Tauri 窗口、托盘、全局快捷键、剪贴板、系统通知和外部链接能力。
- 修复透明窗口下白屏、空窗口、主题未立即生效和 Linux 图标显示异常等问题。
- Linux 支持标准 XDG 配置路径，默认使用 `~/.config/copytranslator`。
- 应用内更新检查改为读取当前仓库 Release：`wang-borong/CopyTranslator`。

### 界面与交互

- 优化主界面阅读区，改善原文输入区、译文显示、备用引擎提示、字体颜色和小窗口布局。
- 设置界面支持可调整宽度，减少窄屏下配置项被挤压的问题。
- 支持浅色、深色、跟随系统主题，并使主题切换即时生效。
- 支持窗口透明度配置，透明窗口在支持合成器的桌面环境中可见底层窗口。
- 语言显示改为本地化名称，中文界面显示“英语/中文”，英文界面显示“English/Chinese”。
- 增加软件语言设置，减少界面中英文混杂。

### 翻译与 AI

- 完善翻译器管理，修复翻译后引擎列表丢失的问题。
- 增强 AI 翻译引擎配置，支持 OpenAI 兼容接口、供应商模板、模型启用和高级提示词设置。
- 支持自定义翻译角色、提示词预设、System Prompt、温度、最大 Token 和格式保留策略。
- 备用引擎提示改为非遮挡式显示，避免小窗口下覆盖译文内容。
- 保留 Google、百度、彩云、有道、搜狗、DeepL、腾讯、阿里云、Azure、火山、Yandex、小牛、StepFun 等翻译引擎。

### OCR 与快捷键

- 集成 OCR 配置入口，支持通过百度 OCR REST API 识别剪贴板图片并继续翻译。
- 增加快捷键配置功能，区分全局快捷键和窗口内快捷键。
- 修复布局按钮与单次增量复制相关交互，使布局切换和增量复制入口更可靠。

### 构建与发布

- 定制 GitHub Actions Release 流程，支持 Windows、macOS、Linux 多平台构建。
- 更新 Tauri、Vue、TypeScript、Rust 依赖，并修复 npm audit 发现的问题。
- 新增版本同步脚本，可一次性更新 npm、Cargo、Tauri 配置和前端常量。

### 升级说明

- 从旧版本升级时，建议先备份原配置文件。
- 旧路径 `~/copytranslator/copytranslator.json` 会在首次启动时尝试迁移到系统配置目录。
- 如果全局快捷键无法注册，通常是被系统或其他软件占用，请在设置中换用其他组合键。
- AI 翻译和 OCR 依赖第三方服务，请确认 API Key、余额、模型权限和网络代理配置。
